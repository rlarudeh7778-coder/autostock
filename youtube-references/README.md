# 유튜브 레퍼런스 수집기 (Outlier Finder)

주제어를 입력하면 관련 유튜브 영상을 수집해, **"채널 평균 대비 유난히 잘 터진 영상(아웃라이어)"** 만
골라 보여주는 CLI 도구입니다.

> **핵심 가설**
> 한 영상의 조회수가 그 채널의 평균보다 훨씬 높다면(아웃라이어), 그건 채널 규모 덕이 아니라
> **그 영상의 주제·제목·썸네일이 먹혔다는 신호**입니다. 즉 "베껴올 가치가 있는 레퍼런스"죠.
> 그래서 이 도구는 **절대 조회수가 아니라 "평소 대비 몇 배 터졌나(아웃라이어 배수)"** 를 핵심 지표로 봅니다.

---

## 설치

```bash
cd youtube-references
pip install -r requirements.txt
```

## API 키 발급

1. [Google Cloud Console](https://console.cloud.google.com/) 접속 → 프로젝트 생성(또는 선택).
2. **API 및 서비스 → 라이브러리** 에서 **YouTube Data API v3** 를 찾아 **사용 설정**.
3. **API 및 서비스 → 사용자 인증 정보 → 사용자 인증 정보 만들기 → API 키** 로 키 생성.
4. (권장) 생성된 키를 **YouTube Data API v3 로만** 사용하도록 제한.

발급받은 키는 다음 중 한 방법으로 등록하세요(코드에 하드코딩 금지):

```bash
# 방법 1) 환경변수
export YOUTUBE_API_KEY="발급받은_키"

# 방법 2) .env 파일
cp .env.example .env
# .env 를 열어 YOUTUBE_API_KEY=... 채우기
```

## 실행

```bash
python main.py "홈카페"
python main.py "재테크 초보" --multiplier 4 --order viewCount --since-months 6
python main.py "퇴사" --format short --max-search 100 --recent 20
```

### 옵션

| 옵션 | 기본값 | 설명 |
|------|--------|------|
| `topic` (필수) | — | 검색할 주제어 |
| `--multiplier` | 4 | 아웃라이어 배수 임계값 (영상 조회수 ≥ 채널 평균 × N) |
| `--order` | relevance | 검색 정렬 (`relevance`/`viewCount`/`date`/`rating`) |
| `--max-search` | 150 | 검색으로 모을 후보 영상 수 |
| `--recent` | 30 | 채널 평균 산정용 최근 영상 수 |
| `--since-months` | (없음) | 최근 N개월 영상만 검색 |
| `--format` | all | `short`(숏폼)/`long`(롱폼)/`all` |
| `--output-dir` | . | CSV/JSON 저장 폴더 |

## 출력

- **콘솔**: 순위 / 제목 / 조회수 / 좋아요 / 댓글 / 아웃라이어 배수 / 일일조회수 / 채널 / 구독자 / 업로드일 / 형식 표
- **파일**: `references_{주제어}_{날짜}.csv` + 동일 이름 `.json` (썸네일 URL 포함 → 제목·썸네일 패턴 분석 가능)
- **패턴 요약**: 제목 앵글(숫자형/질문형/호기심형) 분포, 구독자 규모별 아웃라이어 분포, 숏폼/롱폼 분포

---

## 정확도를 위한 계산 원칙

- **최근 영상 기준 평균**: 채널 전체가 아닌 최근 N개(기본 30) 영상으로 평균을 냅니다. 옛날 영상이 평균을 흐리지 않도록.
- **평균 + 중앙값 병행**: 한 영상의 떡상으로 평균이 부풀려지는 문제를 완화하기 위해 중앙값 기준 배수(`outlier_median`)도 함께 저장합니다.
- **숏폼/롱폼 분리**: 영상 길이(60초 이하 = 숏폼)로 나눠, **같은 형식끼리만** 평균을 비교합니다. 숏폼과 롱폼은 조회수 스케일이 완전히 다르기 때문입니다.
- **신뢰도 표시**: 형식별 비교 표본이 5개 미만이면 `⚠`(low_confidence)로 표시합니다.

## 보조 지표

- **아웃라이어 배수** = 영상 조회수 ÷ 채널 평균 조회수 (핵심 필터·정렬 기준)
- **일일 조회수(velocity)** = 조회수 ÷ 경과일 → 최근 가속 중인 영상 포착
- **참여율** = (좋아요 + 댓글) ÷ 조회수 → 클릭을 넘어선 실제 반응
- **구독자 대비 조회수** = 조회수 ÷ 구독자 수

---

## 운영 원칙 (결과 해석 기준)

1. **아웃라이어 배수 > 채널 규모.** 조회수 100만이어도 채널 평균이 80만이면 평범한 영상. 조회수 5만이어도 평균이 5천이면 10배 터진 진짜 레퍼런스.
2. **작은 채널의 아웃라이어를 놓치지 마라.** 구독자 후광 없이 순수하게 주제·제목·썸네일로 이긴 케이스 → 가장 베낄 가치가 큼. (구독자 규모별 분포로 확인)
3. **최근성을 본다.** `--since-months` 로 최근 수요를 반영한 영상만 볼 수 있음.
4. **일일 조회수로 "지금 뜨는" 영상을 잡는다.** 누적 조회수는 오래된 영상에 유리하므로 velocity를 함께 본다.
5. **제목과 썸네일이 진짜 상품이다.** 제목·썸네일 URL을 수집하고 제목 패턴별로 묶어 보여준다.
6. **참여율로 진짜 반응을 검증한다.** 조회수만 높고 좋아요·댓글이 빈약하면 낚시일 수 있다.
7. **수집은 분류로 이어진다.** 리스트가 아니라 "어떤 앵글/포맷이 반복적으로 터지는지" 패턴을 본다.

## 쿼터 주의

- YouTube Data API 기본 일일 쿼터는 **10,000 유닛**.
- `search.list` 는 **1회당 100 유닛**으로 비쌉니다. `videos.list`/`channels.list`/`playlistItems.list` 는 1 유닛.
- 이 도구는 검색은 필요한 페이지만 돌고, 상세 조회는 항상 50개씩 배치로 묶어 쿼터를 아낍니다.
- 그래도 `--max-search` 를 크게 잡으면 검색 유닛이 늘어납니다. (예: 150개 = search 3페이지 = 300 유닛)
- **쿼터 초과 시**에도 그 시점까지 수집·분석된 부분 결과를 CSV/JSON으로 저장하고 종료합니다.

## 참고

- 좋아요/댓글이 비공개인 영상은 `0`이 아니라 `비공개`(null)로 구분 표시합니다.
- 구독자 수를 숨긴 채널은 구독자 관련 지표가 `null` 로 처리됩니다.
